Voronoi Diagram

Voronoi diagram is a computational geometry concept that represents partition of a given space onto regions, with bounds determined by distances to a specified family of objects. The application area of this concept vary from Archaeology to Zoology. Boost.Polygon provides implementation of Voronoi diagram data structure in 2D space. Internal representation consists of a three arrays, that respectively contain: Voronoi cells (area around input sites bounded by Voronoi edges), Voronoi vertices (points where three or more Voronoi edges intersect), Voronoi edges (one dimensional curves containing points equidistant from the two closest input sites). Each of the primitives (cell, vertex, edge) contains pointers to other linked primitives, so that it's always possible to efficiently traverse Voronoi diagram. Picture below shows Voronoi vertices in red, Voronoi edges in black, input sites that correspond to the Voronoi cells in blue (it is considered that each input segment consists of three sites: segment itself and two endpoints). As one may notice each segment is split onto segment site itself and both its endpoints. As the result two additional Voronoi edges are constructed per each segment. This is made to simplify representation of the Voronoi diagram.



Voronoi diagram declaration and list of member functions is following:

template <typename T, typename TRAITS = voronoi_diagram_traits<T> >
class voronoi_diagram;
voronoi_diagram()
 Default constructor.
void clear()
 Clears diagram.
const cell_container_type &cells() const
 Returns const reference to Voronoi cells container.
const vertex_container_type &vertices() const
 Returns const reference to Voronoi vertices container.
const edge_container_type &edges() const
 Returns const reference to Voronoi edges container.
unsigned int num_cells() const
 Returns number of cells in the Voronoi diagram.
This value should be the same as the size of cells container.
unsigned int num_edges() const
 Returns number of edges in the Voronoi diagram.
This value should be twice smaller then the size of edges container.
The reason is that we have two half-edges for each Voronoi edge.
unsigned int num_vertices() const
 Returns number of vertices in the Voronoi diagram.
This value should be the same as the size of vertices container.

Voronoi Edge

Voronoi edge is represented as a bit improved classical half-edge data structure. The declaration and list of member functions is following:

template <typename T>
class voronoi_edge;
voronoi_edge()
 Default constructor.
voronoi_cell_type *cell()
 Returns pointer to the Voronoi cell that edge belongs to.
const voronoi_cell_type *cell() const
 Returns const pointer to the Voronoi cell that edge belongs to.
void cell(voronoi_cell_type *c)
 Sets Voronoi cell pointer for the cell current edge belongs to.
voronoi_vertex_type *vertex0()
 Returns pointer to the start point of the edge.
If edge is infinite in that direction returns NULL.
const voronoi_vertex_type *vertex0() const
 Returns const pointer to the start point of the edge.
If edge is infinite in that direction returns NULL.
void vertex0(voronoi_vertex_type *v)
 Sets start point pointer of the edge.
voronoi_vertex_type *vertex1()
 Returns pointer to the end point of the edge.
If edge is infinite in that direction returns NULL.
const voronoi_vertex_type *vertex1() const
 Returns const pointer to the end point of the edge.
If edge is infinite in that direction returns NULL.
void vertex1(voronoi_vertex_type *v)
 Sets end point pointer of the edge.
voronoi_edge_type *twin()
 Returns pointer  to the twin edge.
const voronoi_edge_type *twin() const
 Returns const pointer to the twin edge.
void twin(voronoi_edge_type *e)
 Sets twin edge pointer of the edge.
voronoi_edge_type *next()
 Returns pointer to the CCW next edge within the corresponding Voronoi cell.
Edges not necessarily share a common vertex.
const voronoi_edge_type *next() const
 Returns const pointer to the CCW next edge within the corresponding Voronoi cell.
Edges not necessarily share a common vertex.
void next(voronoi_edge_type *e)
 Sets CCW next edge pointer of the edge.
voronoi_edge_type *prev()
 Returns pointer to the CCW prev edge within the corresponding Voronoi cell.
Edges not necessarily share a common vertex.
const voronoi_edge_type *prev() const
 Returns const pointer to the CCW prev edge within the corresponding Voronoi cell.
Edges not necessarily share a common vertex.
void prev(voronoi_edge_type *e)
 Sets CCW prev edge pointer of the edge.
void *data() const
 Returns pointer to the data associated with the edge.
void data(void *d) const
 Sets data pointer of the edge.
This allows user to associate any data type with the edge.
voronoi_edge_type *rot_next()
 Returns pointer to the CCW next edge rotated around the edge start point.
If edge is infinite in that direction returns NULL.
const voronoi_edge_type *rot_next() const
 Returns const pointer to the CCW next edge rotated around the edge start point.
If edge is infinite in that direction returns NULL.
voronoi_edge_type *rot_prev()
 Returns pointer to the CCW prev edge rotated around the edge start point.
If edge is infinite in that direction returns NULL.
const voronoi_edge_type *rot_prev() const
 Returns const pointer to the CCW prev edge rotated around the edge start point.
If edge is infinite in that direction returns NULL.
bool is_finite() const
 Returns true if both endpoints of the edge are finite, else false.
bool is_linear() const
 Returns true if edge is linear (segment, ray, line), else false.
bool is_curved() const
 Returns true if edge is curved (parabolic arc), else false.
bool is_primary() const
 Returns false if the edge goes through segment endpoint, else true.

All the above methods have O(1) complexity. The size of the Voronoi edge structure is equal to: 6 * sizeof(void *).

Voronoi Cell

Voronoi cell is represented by a site the cell contains and a pointer to the incident edge. The declaration and list of member functions is following:

template <typename T>
class voronoi_cell;
voronoi_cell(const point_type &p1,
             voronoi_edge_type *edge)
 Constructs Voronoi cell from the given point site and pointer to the one of boundary edges.
voronoi_cell(const point_type &p1,
             const point_type &p2,
             voronoi_edge_type *edge)
 Constructs Voronoi cell from the given segment site and pointer to one of boundary edges.
const point_type &point0() const
 If cell contains point site returns it.
If cell contains segment site returns its first endpoint.
const point_type &point1() const
 If cell contains point site returns it.
If cell contains segment site returns its second endpoint.
voronoi_edge_type *incident_edge()
 Returns pointer to one of the boundary edges.
const voronoi_edge_type *incident_edge() const
 Returns const pointer to one of the boundary edges.
void incident_edge(voronoi_edge_type *e)
 Sets incident boundary edge pointer of the cell.
void *data() const
 Returns pointer to the data associated with the cell.
void data(void *d) const
 Sets data pointer of the cell.
This allows user to associate any data type with the cell.
bool contains_point() const Returns true if cell contains point site, else false.
bool contains_segment() const Returns true if cell contains segment site, else false.
bool is_degenerate() const Returns true if cell doesn't have incident edge.
Could happen if a few input segments share a common endpoint.

All the above methods have O(1) complexity. The size of the Voronoi cell structure is equal to: 2 * sizeof(void *) + 4 * sizeof(coordinate_type).

Following code snipped effectively traverses Voronoi edges around the Voronoi cell:

const voronoi_edge<double> *edge = cell->incident_edge();
do {
  edge = edge->next();
  // Do smth. with edge.
} while (edge != cell->incident_edge());

Voronoi Vertex

Voronoi vertex is represented by a point that corresponds to the vertex and a pointer to the incident edge. The declaration and list of member functions is following:

 Constructs Voronoi vertex that corresponds to the give point and incident edge.
const point_type &vertex() const
 Returns point that represents vertex.
voronoi_edge_type *incident_edge()
 Returns incident edge pointer.
const voronoi_edge_type *incident_edge() const
 Returns const pointer to the incident edge.
void incident_edge(voronoi_edge_type *e)
 Sets incident edge pointer.
void *data() const
 Returns pointer to the data associated with the vertex.
void data(void *d) const
 Sets data pointer of the cell.
This allows user to associate any data type with the vertex.

All the above methods have O(1) complexity. The size of the Voronoi vertex structure is equal to: 2 * sizeof(void *) + 2 * sizeof(coordinate_type).

Following code snipped effectively traverses Voronoi edges around the Voronoi vertex:

const voronoi_edge<double> *edge = vertex->incident_edge();
do {
  edge = edge->next();
  // Do smth. with edge.
} while (edge != vertex->incident_edge());

Voronoi Diagram Traits

Voronoi diagram traits are used to configure Voronoi diagram data structure. The declaration and list of required type definitions is following:

template <typename T>
struct voronoi_diagram_traits;
coordinate_type
 The main coordinate type of the Voronoi diagram internal structures.
ctype_converter_type
 Coordinate type converter structure.
Converts coordinates provided by Voronoi builder to the internal coordinate type.
point_type
 2D point data structure.
cell_type
 Voronoi cell type.
vertex_type
 Voronoi vertex_type.
edge_type
 Voronoi edge_type.
vertex_equality_predicate_type
 Predicate that returns true if two points are considered to be equal.
This is used to unite nearby Voronoi vertices.
