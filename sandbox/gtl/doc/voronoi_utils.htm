Voronoi Utils

Voronoi utilities implements a set of tools that users may find useful especially for the visualization of Voronoi diagrams or discretization of parabolic edges. Keep in mind that there are no warranties about precision of the output structures produces by utils class. This is mostly because input coordinate type utilities operate with is double. Any degeneracies may be reported, however those are of low priority to be fixed. Class declaration is following:

template <typename T, typename TRAITS = voronoi_utils_traits<T> >
class voronoi_utils;

T - output coordinate type used by the utilities.
TRAITS - output geometries type traits (see definition below).

Member Functions

template <typename CT>
static brect_type scale_bounding_rectangle(
    const bounding_rectangle<CT> &brect,
    fpt_type factor = 1.0)
 Returns scaled bounding rectangle.
The center of transformation corresponds to the center of the input rectangle.
template <typename CT>
static void discretize_finite_edge(
    const voronoi_edge<CT> &edge,
    coordinate_type max_error,
    point_set_type &discretization)
 Provides point discretization of the input voronoi edge.
If the edge is infinite (ray, line) doesn't fill output point set.
Max error specifies maximum distance that is allowed between original parabolic arc and its approximation.
template <typename CT>
static void clip_linear_edge(
    const voronoi_edge<CT> &edge,
    const brect_type &brect,
    point_set_type &clipped_edge)
 Clips the input Voronoi edges with specified rectangle.
If the edge is a parabolic arc doesn't fill output point set.
template <typename PointType>
static void clip_linear_edge(
    const PointType &p1,
    const PointType &p2,
    const brect_type &brect,
    point_set_type &clipped_edge)
 Clips the input linear edge with specified rectangle.
Edge is defined by its endpoints.

Bounding Rectangle

The module provides implementation of a simple bounding rectangle structure with following declaration:

template <typename T>
class bounding_rectangle;

T - coordinate type rectangle operates with.

Member Functions

bounding_rectangle()
 Default constructor. Initializes empty bounding rectangle.
bounding_rectangle(
    coordinate_type x1,
    coordinate_type y1,
    coordinate_type x2,
    coordinate_type y2)
 Constructs bounding rectangle with given coordinates.
void update(
    coordinate_type x,
    coordinate_type y)
 Extends bounding rectangle with a specified point defined by its coordinates.
If the rectangle is empty sets coordinates of the bottom left and top right corners.
bool is_empty() const
 Returns true if rectangle is empty (there were no updates), else false.
void clear()
 Sets rectangle to an empty state.
bool contains(
   coordinate_type x,
   coordinate_type y) const
 Returns true if rectangle contains given point defined by its coordinates, else false.
coordinate_type x_min() const
 Returns x coordinate of the bottom left corner of rectangle.
coordinate_type y_min() const
 Returns y coordinate of the bottom left corner of rectangle.
coordinate_type x_max() const
 Returns x coordinate of the top right corner of rectangle.
coordinate_type y_max() const
 Returns y coordinate of the top right corner of rectangle.

Voronoi Utils Traits

Below is shown the default Voronoi utilities traits:

template <typename fpt_>
struct voronoi_utils_traits;

template<>
struct voronoi_utils_traits<double> {
    typedef double fpt_type;
    typedef detail::point_2d<fpt_type> point_type;
    typedef std::vector<point_type> point_set_type;
    typedef bounding_rectangle<fpt_type> brect_type;
    typedef struct {
        template <typename CT>
        fpt_type operator()(const CT& value) const {
            return static_cast<fpt_type>(value);
        }
    } ctype_converter_type;
};

Most of the types define output geometries. ctype_converter_type is used to cast coordinate type of the input primitives to the one used by utils internally.
