Macro BOOST_LOCAL_FUNCTION_END

BOOST_LOCAL_FUNCTION_END — This macro is used to specify the local function name.

Synopsis

// In header: <boost/local_function.hpp>

BOOST_LOCAL_FUNCTION_END(qualified_function_name)

Description

This macro must follow the local function body code block { ... } as shown in the BOOST_LOCAL_FUNCTION_PARAMS documentation.

The local function name can be prefixed by the "keyword" inline (see the {Advanced_Topics, Advanced Topics} section): 

  ... BOOST_LOCAL_FUNCTION_NAME(inline name)

This increases the chances that the compiler will be able to inline the local function calls (thus reducing their run-time). However, inlined local functions cannot be passed as template parameters or assigned to other functors like boost::function (that is true on C++03 compilers, see BOOST_LOCAL_CONFIG_COMPLIANT, but inlined local functions can instead be passed as template parameters on C++11 compilers). On C++11 compilers, there is no need to declare a local function lined because this library will automatically use C++11 specific features to inline the local function while always allowing to pass it as a template parameter.

The local function name can also be prefixed by the "keyword" recursive (see the {Advanced_Topics, Advanced Topics} section): 

  ... BOOST_LOCAL_FUNCTION_NAME(recursive name)

This allows the local function to recursively call itself from its body (as usual in C++). However, compilers have not been observed to be able to inline recursive local function calls (not even when the recursive local function is also declared inlined).

Parameters:

name The name of the local function. The local function name can be prefixed by the "keyword" inline to declare the local function inlined or by the "keyword" recursive to declare the local function recursive (see the {Advanced_Topics, Advanced Topics} section). The local function name cannot be the name of an operator operator... and it cannot be the same name of another local function declared within the same enclosing scope (use the boost::local::function::overload functor to overload local functions, see the {Advanced_Topics, Advanced Topics} section).

See: BOOST_LOCAL_FUNCTION_PARAMS, Tutorial section, {Advanced_Topics, Advanced Topics} section, BOOST_LOCAL_CONFIG_COMPLIANT.
