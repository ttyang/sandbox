has_stable_iterators_selector

Synopsis 

namespace boost {

    template <typename Selector>
    struct has_stable_iterators_selector
    {
        // typedef ... type;
    };
}  // namespace boost

Description

Returns boost::mpl::true_ if the type to be returned by container_gen does not invalidate iterators other than the argument during an erase() call, or if the erase() method does not exist, boost::mpl::false_ otherwise.

The current implementation returns boost::mpl::false_ when passed in selectors for which container_gen will return a Hashed Associative Container, a Flat Associative Container, or a Random Access Container other than boost::array or boost::container::stable_vector; it will return boost::mpl::true_ for all other types of selectors. Specialize this metafunction to return boost::mpl::false_ for your custom selector if and only if the type to be returned by container_gen invalidates iterators other than the argument during an erase() call.

Where defined 

#include <boost/container_gen/has_stable_iters_selector.hpp>
