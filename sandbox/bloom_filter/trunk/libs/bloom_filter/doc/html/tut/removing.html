Removing Elements

Element removal is an operation supported only by the counting Bloom filters. remove can introduce false negatives, so it must be used with caution. The list below highlights the possibilities:

  • underflow: element removal caused a bin to go from 0 to BIN_MAX.
  • overflow: element insertion caused a bin to go from BIN_MAX to 0.

This implementation does not check for either error.

counting_bloom_filter<int, 1000, 2> bloom; bloom.insert(1); bloom.probably_contains(1); // true bloom.remove(1); bloom.probably_contains(1); // false bloom.insert(1); // bin[1] = 01 bloom.insert(1); // bin[1] = 10 bloom.insert(1); // bin[1] = 11 bloom.insert(1); // bin[1] = 00 bloom.probably_contains(1); // overflow, false bloom.remove(1); // bin[1] = 11 bloom.probably_contains(1); // underflow, true

Range removal is also supported for counting Bloom filter types.
