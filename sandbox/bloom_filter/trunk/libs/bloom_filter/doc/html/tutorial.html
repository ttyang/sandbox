Overview

There are two primitive operations that are supported by all Bloom filters: insert and probably_contains. A third operation, remove, is supported by counting Bloom filters, and a few similar variants. In order to control the false positive rate of a Bloom filter, you can control the number of hash functions used, as well as the bit capacity.

Types of Bloom Filters Provided

  • Basic Bloom filter (basic_bloom_filter, dynamic_bloom_filter)
  • Counting Bloom filter (counting_bloom_filter)

Basic Bloom Filter

To use this Bloom filter, there are two template parameters to be aware of and two functions to utilize:

  • Template parameters:
    • Type: the type of elements this Bloom filter will store
    • Size: the number of bits used to store the Bloom filter state
  • Basic functions:
    • void insert(const T&);
    • bool probably_contains(const T&);
  • Advanced template parameters:
    • HashFunctions: a boost::mpl::vector of Hasher objects.
  • Advanced state monitoring functions:
    • size_t size() const: returns the value of the template parameter Size
    • size_t num_hash_functions() const: returns the number of hash functions used to define this Bloom filter.
    • double false_positive_rate() const : returns the current false positive rate

All the details of insertion and querying the Bloom filter are handled transparently by the insert() and probably_contains() functions, regardless of the type, bit size, and hash functions used. This makes it very convenient to experiment with different Bloom filter parameters in projects you commit to.
