A counting Bloom filter implemented using the two-hash technique. It is best used when:

  • Storage requirements do not exceed stack space.
  • You wish to generate many hash values convieniently.
  • Element removal is necessary.

Template Parameters Reference

T
The type used for all Bloom filter operations.
NumBins
The number of bins to use for storage.
BitsPerBin
The number of bits to use per bin.
HashValues
The number of hash values to generate.
ExpectedInsertionCount
Currently unused. See here.
HashFunction1
The first hash function used to generate hash values.
HashFunction2
The second hash function used to generate hash values.
ExtensionFunction
A function that modifies hash values generated additively.
Block
Used to determine the granularity of bit operations.
