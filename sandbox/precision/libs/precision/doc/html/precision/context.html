The context among existing implementations

Many existing implementations already support float, double, and long double. In addition, some of these either are or strive to be compliant with IEEE_ floating-point format. In these cases, it will be straightforward to support (at least) a subset of the proposed optional floating-point typedefs having specified width by adding any desired optional type definitions and the corresponding macro definitions.
