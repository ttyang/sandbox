Binary Tree Node

Description

Models of this concept store at most two child nodes: left and right. The operations that this concept defines include access to these children and rotations that use the appropriate child as a pivot node.

Refinement of

Non-Associative Tree Node

Notation

Node

A type that models the Binary Tree Node concept.

node

An object of type Node::pointer.

const_node

An object of type Node::const_pointer.

data

An object of type Node::traits::data_type const&.

Associated Types

No additional types beyond those defined in the Non-Associative Tree Node concept.

Rotate left

node->rotate_left()

Node::pointer

Rotates the tree node in its hierarchy so that its right child, the pivot node, becomes its parent. The left child of the pivot node becomes the right child of the tree node. Returns the pivot node.

Amortized constant

Rotate right

node->rotate_right()

Node::pointer

Rotates the tree node in its hierarchy so that its left child, the pivot node, becomes its parent. The right child of the pivot node becomes the left child of the tree node. Returns the pivot node.

Amortized constant
